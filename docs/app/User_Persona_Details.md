# 表示用ペルソナ機能仕様 (User Persona Details)

本ドキュメントでは、チャット時の会話相手からの呼び名やUI上でのアバター表現、およびメモ機能における編集・共有対象として利用される**「表示用ペルソナ（Persona）」**について、機能と内部仕様を解説します。

---

## 1. 表示用ペルソナ (Persona) の定義と役割

表示用ペルソナ（Persona）は、チャット内のメッセージ送信者としての「見かけ上のプロファイル」を提供する UI レイヤーの概念です。

*   **スコープ**: ログイン中の認証ユーザー (Identity) の LocalStorage 名前空間内 (`${authUsername}_chat_personas`) にリストとして保持されます。
*   **主な役割**:
    1.  チャット対話時のユーザー側発信者名およびアバター絵文字の決定。
    2.  ユーザーメモ作成時の「作成者」「更新者」としての記録。
    3.  ユーザーメモを誰に公開するかを指定する「対象ターゲット（オーディエンス）」の選択肢。

---

## 2. 権限（ロール）によるペルソナ管理の制限

ペルソナは、ログインしている認証ユーザーのシステム権限（`role`）に応じて、管理・作成できる範囲が厳格にコントロールされます。

### A. 一般ユーザー (`role === 'user'`) の場合
*   **作成制限**: 新たなペルソナを追加することはできません。
*   **初期状態**: 認証ユーザー自身（ログイン情報と同一の名前、ロール、アバター）を表す**1つのペルソナのみ**がリストに登録されます。
*   **UI挙動**: 切り替え用の UI は非表示、または選択肢が1つのみに制限されます。

### B. 管理者ユーザー (`role === 'admin'`) の場合
*   **追加機能**: 設定タブの「Persona Settings & Switching」にある「ペルソナ管理パネル」から、新しい表示用ペルソナを自由に何個でも追加登録できます。
*   **設定項目**: 名前（任意の表示名）、アバター絵文字（UIに用意されたグリッドから選択）、権限ロール（`admin` または `user`）。
*   **削除機能**: 不要になったペルソナを削除できます。ただし、現在選択中（アクティブ）のペルソナについては、誤操作によるエラーを防ぐため**削除ボタンが無効化**されます。

---

## 3. UI/UXにおけるペルソナの切り替えとレンダリング

### A. 切り替え処理 (Switching)
設定画面の「Switch Chat Persona」セクションで任意のペルソナカードをクリックすると、`switchPersona(id)` アクションが走り、フロントエンドの状態管理である `activePersona` シグナルが更新されます。
*   *注意*: これは UI 表示上の切り替えであり、ログイン中の認証セッション（Identity）自体が変わるわけではないため、セキュリティスコープは維持されます。

### B. チャット画面での描画
*   **Header バー**: 現在アクティブなペルソナのアバター絵文字と名前が、ログインステータスとして Header の右側にリアルタイム表示されます。
*   **メッセージバブル**: 送信されたメッセージの横に、そのメッセージを送信した時点でのアクティブなペルソナのアバター絵文字が割り当てられます。

---

## 4. ユーザーメモ（User Memos）との密接な連携

ペルソナ機能は、サイドバーで管理される「ユーザーメモ」のメタデータとして重要な役割を果たします。

### A. 自動メタデータ記録
メモを新規作成、または既存のメモを編集・保存した際、バックエンドへの保存データに以下の情報が自動的にセットされます。
*   **作成者 (`creator`)**: 新規作成時のアクティブなペルソナ名。
*   **更新者 (`updater`)**: 最後に保存した時点でのアクティブなペルソナ名。

*UI上の制約*: 誤ったペルソナ設定や他のペルソナ・エージェントを誤って手動選択するのを防ぐため、メモ作成フォーム（Create New Memo）およびメモカード（Memo Card）のUIからは「作成者」「更新者」の選択ドロップダウンは非表示（排除）になっています。代わりに、メタデータとして「Creator」「Updater」が読み取り専用（Read-only）のテキストとして自動表示されます。

### B. 公開対象（ターゲットオーディエンス）の動的決定
各メモには、そのメモを適用・公開する相手を選択するためのチェックボックスグループ（Target Audiences）が UI に表示されます。
*   このチェックボックスの選択候補リストには、現在 `LocalStorage` のペルソナリスト（`chat_personas`）に登録されているすべてのペルソナ名が動的に抽出されて表示されます。
*   選択されたターゲット情報は SQLite データベースの `memo_audiences` テーブルに保存され、AI エージェントがメモを参照する際の公開スコープ判断の材料になります。
